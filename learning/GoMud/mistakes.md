## 错误 #1 — Go 内嵌的字段/方法提升机制

**日期**: 2026-07-14
**场景**: 第二幕思考题 — Player 内嵌 Entity 后能否直接调用 TakeDamage()

### 我的错误回答
> "不行，因为 Entity 嵌入 Player 以后，它变成了 Player 直接的 Name HP MaxHP，而不再是 Entity 中的，不应该再通过 Entity 来进行赋值"

### 错误原因
- 混淆了"字段提升"和"字段消失"
- 误以为内嵌后 Entity 作为独立结构就不存在了
- 没有理解"提升"是"同时存在于两个层级"，而非"从内层移到外层"

### 正确理解
```
内嵌不是删除，而是提升。

内嵌的字段和方法像透明的层：
  - 从外面看：可以直接触及（p.Name, p.TakeDamage()）
  - 从里面看：也依然完整存在（p.Entity.Name, p.Entity.TakeDamage()）

两种写法完全等价，指向同一块内存。
```

### 验证方式
```go
p.Name == p.Entity.Name        // true，同一块内存
p.HP  == p.Entity.HP           // true，TakeDamage 修改的就是内嵌 Entity
p.IsAlive()                    // 合法，方法也被提升
```

## 备注纠错 — Q1 诊断题

**日期**: 2026-07-14
**场景**: 造物主试炼 Q1 — p1 与 p2 的区别

### 我的回答
> "p2 指向了一个已经存在的 Creature 实例的内存地址"

### 纠正
`p2 := &Creature{HP: 100}` 中的 `&Creature{HP: 100}` 是"创建新的 + 取地址"一步到位，不是指向"已经存在的"实例。p2 是 `*Creature` 类型（指针），p1 是 `Creature` 类型（值）。
